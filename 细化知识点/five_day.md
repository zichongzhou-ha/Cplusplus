### 为什么多态必须指针 / 引用调用？

对象调用 = 拷贝 = 切片 = 子类信息丢失 = 无法多态

指针 / 引用调用 = 不拷贝 = 不切片 = 保留完整对象 = 能查虚表 = 多态


----

### 为什么构造函数不能是虚函数？为什么析构函数最好写成虚函数？

1. 虚函数依赖虚表指针（vfptr）
    - 要实现多态，必须通过对象里的 vfptr 去查虚表。
2. 构造函数的作用是初始化对象
    - 构造函数执行时，对象正在被创建，还没完全构造好。
    - 虚表指针是在构造函数里才被赋值的！
3. 执行顺序：
    - 分配内存
    - 进入构造函数 → 先给 vfptr 赋值
    - 再执行你写的代码
4. 也就是说：
构造函数执行前半段，vfptr 还不存在 / 还没指向正确虚表。
如果构造函数是虚函数
调用时需要查虚表 → 但虚表指针还没初始化 → 逻辑死循环

5. 析构函数建议写成虚函数
    - 用父类指针删除子类对象时，非虚析构只会调用父类析构，造成子类资源泄漏，虚析构才能触发多态，保证子类析构也被调用

### 动态绑定 静态绑定
首先，其实对象里函数的地址是已经确定了，非虚函数的地址就在函数符号表里，对于调用的语句，实际编译阶段就在函数符号表里确定了它的地址，在运行时直接执行就行了，而对于虚函数，虽然对象里的虚函数地址在虚函数表里已经确定了，但是，对于调用的语句，编译时它识别到了这是个虚函数，不能直接从符号表里找，只能等运行时对象的创建之后明确是谁的虚函数，到谁的虚函数表里找，也就是运行时确定地址后顺便执行

### lowerbound upperbound

- lower_bound /upper_bound 不是容器成员，是通用算法函数（底层就是二分），
但它们只对「已经有序」的区间有效！
- [lower_bound, upper_bound)

```
#include <algorithm>
#include <vector>
using namespace std;

int main() {
    vector<int> v = {1,2,3,4,4,4,5};

    // 第一个 >=4
    auto it1 = lower_bound(v.begin(), v.end(), 4);
    // 第一个 >4
    auto it2 = upper_bound(v.begin(), v.end(), 4);

    // 输出所有等于4的元素
    for (auto it = it1; it != it2; ++it) {
        cout << *it << " ";   // 4 4 4
    }
}
```

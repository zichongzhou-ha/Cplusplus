### 声明 定义 初始化
- 声明：告诉编译器 “有这个东西”，但不分配空间
```
extern int a;      // 变量声明

void func();        // 函数声明

class A;            // 类的前置声明
```

- 定义：真正创建这个东西，分配内存
```
int a;              // 变量定义（分配4字节）

void func() { }     // 函数定义（生成指令）

class A { };        // 类定义

A aa;               // 对象定义
```

- 初始化：给变量第一次赋值
```
int a = 10;         // 定义 + 初始化

int b(20);          // 初始化（C++）

A aa(1);            // 调用构造函数初始化
```

总结
```
extern int x;   // 声明：告诉有x，不分配空间
int x;          // 定义：分配空间
x = 10;         // 赋值（不是初始化！）
int x = 10;     // 定义 + 初始化
```
### 内部类
- 内部类作用域在外部类内，使用要加 Outer::(但前提是得设为public，才可以用作用域访问)
- 内部类可访问外部类所有成员（包括 private），但要通过外部类对象 / 指针 / 引用
- 外部类不能访问内部类私有成员
- 两者对象独立，内存独立，生命周期无关
- 内部类可直接访问外部类 static
- 内部类可设为 public/private/protected



### 内存泄露问题 malloc realloc calloc new
malloc/free和new/delete的区别
1. malloc和free是函数，new和delete是操作符
2. malloc申请的空间不会初始化，new可以初始化
3. malloc申请空间时，需要手动计算空间大小并传递，new只需在其后跟上空间的类型即可， 
如果是多个对象，[]中指定对象个数即可
4. malloc的返回值为void*, 在使用时必须强转，new不需要，因为new后跟的是空间的类型
5. malloc申请空间失败时，返回的是NULL，因此使用时必须判空，new不需要，但是new需
要捕获异常
6. 申请自定义类型对象时，malloc/free只会开辟空间，不会调用构造函数与析构函数，而new
在申请空间后会调用构造函数完成对象的初始化，delete在释放空间前会调用析构函数完成
空间中资源的清理

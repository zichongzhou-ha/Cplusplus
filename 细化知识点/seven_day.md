### RAII
>资源在对象构造时获取，对象析构时自动释放。
离开作用域 → 自动释放资源，不用你手动写。

1. 管内存 → 用 智能指针
new 出来的对象不想手动 delete,防止内存泄漏,选择规则：
    - 独占所有权 → unique_ptr（优先用）
    - 共享所有权 → shared_ptr

2. 管锁 → 用 lock_guard / unique_lock

多线程访问共享数据,防止忘记 unlock,防止死锁,选择规则：
- 简单加锁，全程不解锁 → lock_guard
- 需要中途解锁、条件变量 → unique_lock

3. 最精炼总结

>内存泄漏：因为忘记 delete
→ 用 智能指针，自动释放内存。
死锁 / 锁不释放：因为忘记 unlock
→ 用 lock_guard/unique_lock，自动解锁。
两者原理完全一样，都是 RAII：
栈对象离开作用域自动析构，资源自动释放。
一句话区分：
管内存用智能指针
管锁用锁包装器

### 智能指针

一、核心思想：RAII

- 资源在构造时获取，析构时自动释放
- 离开作用域 → 自动释放 → 杜绝内存泄漏、忘记释放锁等问题

二、4 个智能指针（现代 C++ 只需要掌握前 3 个）

1. std::unique_ptr（默认首选、最常用）
- 独占所有权：同一时间只能有一个指针管理对象
- 不能拷贝，只能移动（std::move）
- 最轻、最快、无额外开销
- 适用：绝大多数需要动态分配的场景

2. std::shared_ptr（共享所有权）
- 多个指针可以指向同一个对象
- 内部有引用计数
- 拷贝 → 计数 + 1
- 销毁 → 计数 - 1
- 计数 = 0 才真正释放对象

3. std::weak_ptr（解决 shared_ptr 循环引用）
- 只观察，不拥有
- 不增加引用计数

4. std::auto_ptr（废弃！C++98 遗物，千万别用）
- 拷贝会偷所有权，极其危险
- C++11 被 unique_ptr 取代
- C++17 直接删除

三、make_shared /make_unique（必须会用）
- 创建 unique_ptr → make_unique
- 创建 shared_ptr → make_shared

好处：
- 异常安全，不会内存泄漏
- 效率更高（make_shared 一次分配对象 + 计数）
- 代码更简洁

四、重点难点：循环引用
- A 内部有 shared_ptr<B>
- B 内部有 shared_ptr<A>
- 互相指向 → 计数永远无法到 0 → 内存泄漏
- 解决：把其中一个改成 weak_ptr，因为 weak_ptr 不增加计数，可以正常销毁

### 懒汉模式 饿汉模式

一、饿汉模式（Eager）
- 核心：程序一启动就把单例对象造好，不管你用不用。
- 优点
    - 天生线程安全
    - 全局静态变量在程序启动时由系统初始化，不会有多线程竞争问题。
    - 高并发性能好
    - 获取实例时不需要加锁，高并发下速度极快。
    - 实现简单、无坑
    - 没有双检锁、内存序这些复杂问题。
- 缺点
    - 程序启动变慢
    - 如果单例构造很慢（加载大文件、建立连接），会拖慢启动速度。
    - 资源浪费
    - 哪怕整个程序都不用这个单例，它也已经占着内存了。

二、懒汉模式（Lazy）

- 核心：第一次用的时候才创建，延迟加载。
- 优点
    - 延迟加载，启动快
    - 不用就不创建，不影响程序启动速度。
    - 节约资源
    - 没用到就不占用内存。
- 缺点
    - 线程不安全，必须加锁
    - 多线程下会创建多个实例，必须用双检锁 DCL。
    - 高并发性能稍差
    - 第一次获取会竞争锁，高并发场景不如饿汉快。
    - 实现复杂，容易写错
    - 指令重排、内存序、析构释放等都容易出坑。

三、对比

                            饿汉模式	                懒汉模式

            创建时机	    程序启动 (main 前)	    第一次调用 GetInstance

            线程安全	    天然安全	            不安全，必须加锁

            获取性能	    极快，无锁	            第一次略慢，需竞争锁

            启动速度	    慢（初始化早）	        快（延迟初始化）

            资源占用	    可能浪费	            用到才分配

            实现难度	     简单	                复杂（双检锁 / 内存序）

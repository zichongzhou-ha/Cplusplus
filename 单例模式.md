## 单例模式

### 懒汉模式

```

class Singletion
{
public:
	static Singletion* GetInstance()
	{
		if(_pinst==nullptr)//双检查：只有第一次需要上锁
		{//而且，一个
			//_mtx.lock();
			unique_lock<mutex>lock(_mtx);//局部域的目的是为了它的生命周期/这样的话抛异常它也自动解锁了
			if (_pinst == nullptr)
			{
				_pinst = new Singletion;
			}
			//_mtx.unlock();
		}
		return _pinst;
	}
	Singletion(const Singletion& s) = delete;
private:
	Singletion()
	{}
	 static Singletion* _pinst;
	 static mutex _mtx;
};
Singletion* Singletion::_pinst = nullptr;
mutex Singletion::_mtx;
```

### 饿汉模式

```
class Singletion
{
public:
	static Singletion* GetInstance()
	{
		return &_inst;
	}
	Singletion(const Singletion& s) = delete;
private:
	Singletion()
	{}
	 static Singletion _inst;
};
Singletion Singletion::_inst;
//static对象是在main函数之前创建的，这会只有主线程，所以不存在线程安全

```

### 对比

1. 懒汉模式需要考虑线程安全和释放的问题，实现相对复杂，饿汉模式实现简单
2. 懒汉是一种懒加载模式需要时在初始化创建对象，不会影响程序的启动，饿汉模式则相反，程序启动阶段就创建初始化实例对象，会导致程序启动慢，影响体验 
3. 如果有多个单例类，假设有依赖关系(B依赖A)，要求A单例先创建初始化，B单例在创建初始化，那么就不能饿汉，因为无法保证创建初始化顺序。静态对象我们无法保证初始化顺序
4. 总体而言，懒汉更实用一些
